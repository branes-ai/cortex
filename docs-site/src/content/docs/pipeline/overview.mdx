Visual-Inertial Odometry (VIO) fuses a **camera** (rich geometry, slow,
scale-ambiguous if monocular) and an **IMU** (fast, senses gravity, drifts when
integrated alone) into a single *metric* estimate of where the platform is — and,
just as importantly, **how uncertain that estimate is**.

This section documents the pipeline cortex implements, **stage by stage**. Each
stage is treated as a **contract**: a signature, pre-conditions it assumes,
post-conditions/invariants it must guarantee, the native-unit metrics that prove
it, and where the current implementation is known to fall short. The framing
follows the OpenVINS error-state MSCKF — the de-facto reference filter. *Why*
contracts-and-probes rather than editing-and-watching-ATE is the subject of
[Diagnostic Methodology](/pipeline/diagnostic-methodology/).

## The dataflow

The IMU drives a high-rate **propagation** of the mean and covariance; each camera
image triggers a **clone** of the current pose into a sliding window, the
**frontend** tracks features, lost/marginalized features are **triangulated**, and
the resulting reprojection residuals drive the **MSCKF update** that corrects the
whole window. Old clones are **marginalized** out.

## The stages

| | Stage | What it does |
|---|---|---|
| **S0** | [Sensor & calibration models](/pipeline/stages/s0-sensor-models/) | the measurement functions every residual is built from |
| **S1** | [Initialization](/pipeline/stages/s1-initialization/) | bootstrap attitude, biases, gravity, scale + the initial covariance |
| **S2** | [IMU propagation](/pipeline/stages/s2-imu-propagation/) | predict the mean and covariance forward between images |
| **S3** | [State augmentation / cloning](/pipeline/stages/s3-augmentation/) | add the new image's pose to the sliding window |
| **S4** | [Visual frontend](/pipeline/stages/s4-frontend/) | KLT tracking + RANSAC → feature tracks |
| **S5** | [Feature triangulation](/pipeline/stages/s5-triangulation/) | recover the 3-D point a track corresponds to |
| **S6** | [MSCKF update](/pipeline/stages/s6-msckf-update/) | null-space → QR compress → χ² gate → EKF correction |
| **S7** | [SLAM-feature update](/pipeline/stages/s7-slam-features/) | optional persistent landmarks |
| **S8** | [Zero-velocity update](/pipeline/stages/s8-zero-velocity/) | optional stationarity robustness |
| **S9** | [Marginalization](/pipeline/stages/s9-marginalization/) | drop the oldest clone, keep the window bounded |
| **S10** | [Online calibration](/pipeline/stages/s10-online-calibration/) | extrinsics, intrinsics, time offset |

*(All eleven stages S0–S10 are now written — data-backed where a probe is
wired, contract pages for the optional/not-yet-implemented layers.)*

## What "good" looks like

Two properties, measured separately:

- **Accuracy** — the estimated trajectory is close to truth (ATE / RPE).
- **Consistency** — the filter's reported uncertainty is *honest*: when it says
  ±10 cm it is wrong by about 10 cm, not 1 m. This is the safety property, and the
  one a single accuracy number cannot see. Start with
  **[Reading the metrics](/pipeline/metrics/)**.

<Aside type="caution" title="Status">
  The cortex filter is currently **over-confident** on EuRoC — its covariance is
  too small for its true error. The decomposition of *why*, stage by stage, and
  the tracking-issue index are on the
  [Engineering Status](/pipeline/engineering-status/) page.
</Aside>
