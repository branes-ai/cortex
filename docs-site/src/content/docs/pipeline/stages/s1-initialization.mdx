**A filter has to start somewhere.** 
VIO is a highly non-linear estimator. For its Taylor-series linearizations (Jacobians) to remain valid, the filter must be initialized incredibly close to the true initial state.

S1 bootstraps the initial state — attitude, gyro/accel biases, gravity direction, and (for a moving dynamic start) metric scale — **and** seeds the initial covariance $\mathbf{P}_0$ that informs the filter of its starting uncertainty. 

If S1 fails, it introduces two severe failure modes: returning a *confidently wrong* state when the motion cannot observe it, or seeding $\mathbf{P}_0$ too tight on the directions the initialization could not physically observe. The first causes immediate divergence; the second quietly poisons the EKF with structural over-confidence from $t = 0$.

---

## 1. Static Initialization (Gravity-Leveling)

When the vehicle starts stationary on the ground, S1 detects a quiet window by checking that the standard deviation of gyroscope and accelerometer readings is below a strict threshold. It then averages the raw readings over $M$ samples:

$$\bar{\mathbf{a}} = \frac{1}{M} \sum_{i=1}^M 	ilde{\mathbf{a}}_i, \quad \bar{\boldsymbol{\omega}} = \frac{1}{M} \sum_{i=1}^M 	ilde{\boldsymbol{\omega}}_i$$

### A. Gyroscope Bias Seeding
Since the vehicle is stationary ($\boldsymbol{\omega}_{true} = \mathbf{0}$), any measured angular rate is pure bias:
$$\mathbf{b}_g = \bar{\boldsymbol{\omega}}$$

### B. Attitude Seeding from Gravity
Since the accelerometer only senses gravity, the mean specific force $\bar{\mathbf{a}}$ points exactly opposite to gravity. We use this to compute the initial roll and pitch of the vehicle relative to gravity.

We define the $z$-axis of the IMU frame in the world frame as:
$$\mathbf{z}_I = \frac{\bar{\mathbf{a}}}{\|\bar{\mathbf{a}}\|}$$

To build a full orthogonal rotation matrix $\mathbf{R}_W^I$ that maps gravity $\mathbf{g}_W = [0, 0, -g]^T$ to $\bar{\mathbf{a}}$, we use Gram-Schmidt orthonormalization:
1.  Assume a temporary horizontal $x$-axis: $\mathbf{x}_{tmp} = [1, 0, 0]^T$.
2.  Compute the orthogonal $y$-axis: $\mathbf{y}_I = \frac{\mathbf{z}_I 	imes \mathbf{x}_{tmp}}{\|\mathbf{z}_I 	imes \mathbf{x}_{tmp}\|}$.
3.  Compute the true orthogonal $x$-axis: $\mathbf{x}_I = \mathbf{y}_I 	imes \mathbf{z}_I$.
4.  The initial rotation matrix is then:
    $$\mathbf{R}_I^W = \begin{bmatrix} \mathbf{x}_I & \mathbf{y}_I & \mathbf{z}_I \end{bmatrix}$$

Because gravity has zero horizontal components, **static accelerometer readings can only observe the two tilt axes (roll and pitch)**. The heading (**yaw**) is completely unobservable relative to gravity. Yaw is a free gauge direction; we can set it to an arbitrary starting value ($0^\circ$), but the filter must be told that this yaw direction is completely unobserved.

## 2. Dynamic Initialization (Visual-Inertial Alignment)

If the vehicle starts in motion (e.g., hand-launched or already flying), static gravity-leveling is impossible because $\mathbf{a}_W 
eq \mathbf{0}$. S1 must perform a **Visual-Inertial Alignment** over a short sliding window (usually $1 - 2	ext{ seconds}$).

```
[Camera Track (SfM)] ──▶ Recover up-to-scale relative translations (p_c)
                                                                 │
                                                                 ▼  (Match via scale "s")
[IMU Preintegration] ──▶ Double-integrate raw accelerations (s · Δp_imu)
```

1.  **Structure from Motion (SfM)**: The visual frontend tracks features over multiple frames and runs a bundle adjustment (or five-point algorithm) to recover a **set of relative camera translations $\mathbf{p}_{ci}$ up to a scale factor $s$**.
2.  **IMU Preintegration**: The IMU accelerations are double-integrated over the same intervals to compute relative displacement estimates $\Delta \mathbf{p}_{imu,i}$.
3.  **Linear Scale Alignment**: We set up a linear system matching the visual translations to the double-integrated IMU displacements, solving for the **metric scale $s$, initial velocities $\mathbf{v}_i$, and gravity vector $\mathbf{g}_W$**:
    $$\mathbf{R}_{W}^{c_0} \cdot \mathbf{p}_{c_i} = s \cdot \Delta \mathbf{p}_{imu,i} + \mathbf{v}_0 \Delta t_i + \frac{1}{2} \mathbf{g}_W \Delta t_i^2$$

### The Scale-Observability Cliff
Under gentle motion (such as hover or pure rotation), the acceleration vector does not change. In this regime, the system of equations is **ill-conditioned**, and the metric scale $s$ is physically unobservable. 

To prevent the EKF from starting with a confidently wrong metric scale, Cortex runs a **Singular Value Decomposition (SVD) condition check** on the alignment matrix $A$. If the ratio of the maximum to minimum singular values (condition number) exceeds a strict threshold, the dynamic initialization is safely **declined**, forcing the system to wait for sufficient acceleration excitation.

## 3. The Covariance Seed Suspect: Isotropic Over-Confidence

The initial covariance $\mathbf{P}_0$ is a $15 	imes 15$ matrix that represents the filter's starting uncertainty. 

Historically, many VIO implementations (including those generated by AI Code Assistants) initialize $\mathbf{P}_0$ as an **isotropic matrix**:
$$\mathbf{P}_0 = \sigma^2 \mathbf{I}$$
Where every block (roll, pitch, yaw, velocity, position, gyroscope bias, accelerometer bias) gets the same small initial uncertainty (e.g., $\sigma = 5.73^\circ$ for rotation).

This isotropic seed is **mathematically dishonest and a major source of over-confidence**:

| State Block | Physical Observation at Static Init | Isotropic Seed ($\sigma = 5.73^\circ$) | Physically Honest Seed |
| :--- | :--- | :--- | :--- |
| **Roll / Pitch** | **Observed** via gravity-leveling ($\bar{\mathbf{a}}$). | $5.73^\circ$ | **Small** ($\approx 0.1^\circ$ to $0.5^\circ$) |
| **Yaw (Heading)** | **Unobserved** (gravity has no horizontal component). | $5.73^\circ$ | **Huge** ($\approx 180^\circ$ or treated as infinite) |
| **Velocity** | **Observed** (stationary table is $\mathbf{v} = \mathbf{0}$). | $0.1	ext{ m/s}$ | **Small** ($\approx 10^{-4}	ext{ m/s}$) |
| **Accel Bias** | **Weakly Observed** over a static window. | $0.1	ext{ m/s}^2$ | **Large** ($\approx 1.0	ext{ m/s}^2$) |

By claiming that the unobservable yaw heading is known to $\pm5.73^\circ$ at $t = 0$, the isotropic seed injects false information into the EKF. Because the filter "thinks" its heading is highly accurate, it will reject visual measurements that deviate slightly due to camera rotation, leading directly to the **attitude-localized over-confidence** seen on the [Consistency Analysis](/pipeline/consistency-analysis/) page.

| **Pre-conditions** | Gyro/accel variance is below stationary noise threshold (static init); singular-value conditioning of SfM-IMU matrix is below 100 (dynamic init). |
| **Post-conditions & Invariants** | **1. Positive Scale**: Dynamically recovered scale $s$ is strictly positive ($s > 0$) and finite.<br/>**3.
